Wartość w decyzjach

Zarządzanie kodem

Monorepo

  • + współdzielenie
  • - separacji logiki

Multi repo

  • + separacja
  • - współdzielenie

Zarządzanie zależnościami

Wspólne zależności

  • + spójny stack technologiczny
  • - utrzymywanie spójności

Rozdzielone zależności

  • + łatwe zastosowanie nowych technologii w nowych elementach
  • - ujednolicenie i współdzielenie kodu

Testy

Tak

  • + weryfikacja logiki w rosnącym projekcie
  • - utrzymanie

Nie

  • + "dobry kod" nie wymaga testowania :)
  • - debugowanie funkcjonalności

TypeScript - statyczne sprawdzanie typów

Tak

  • + eliminujemy pewną kategorię błędów
  • - pisanie deklaracji

Nie

  • + szybkość implementacji
  • - późne wykrywanie błędów

Linter

Tak

  • + piszemy raz, czytamy wiele razy
  • - nie wszystkim odpowiada

Nie

  • + indywidualizm
  • - commity z reformatem kodu

CI / GIT / IDE

Tak

  • + kontrola nad jakością
  • - utrzymanie skryptów
